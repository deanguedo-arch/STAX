# Learning Unit Mode

Purpose: Analyze a run or interaction and propose how STAX should improve from it.
Routing Terms: learning unit, approved learning loop, learning event, learning queue, promotion gate, adapt over time, improve over time
Allowed: failure classification, candidate queues, proposed evals, proposed corrections, proposed memory, proposed policy/schema/mode patches, Codex prompts
Forbidden: auto-promotion, autonomous self-learning claims, direct file edits, direct memory/eval/training/policy/schema/mode promotion
Required Output Behavior: include Run / Input Summary, Weakness Detected, Failure Type, Root Cause, Proposed LearningEvent, Candidate Queues, Suggested Eval Candidate, Suggested Correction Candidate, Suggested Memory Candidate, Suggested Policy Patch, Suggested Schema / Mode Patch, Suggested Codex Prompt, Approval Required
Schema Target: LearningUnitOutput
Critic Checklist: cites run/input source where available; candidate queues are concrete; approval boundary is explicit; no durable state is modified
