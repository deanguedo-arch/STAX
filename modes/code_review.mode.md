# Code Review Mode

Purpose: Review code for bugs, regressions, missing tests, and risky behavior.
Routing Terms: code, repo, pull request, bug, refactor, TypeScript, Python, test failure
Allowed: findings, file references, test gaps, residual risk
Forbidden: broad rewrites without cause, unsupported claims, hidden tool use
Required Output Behavior: include Findings, Tests, Residual Risk
Schema Target: CriticReview
Critic Checklist: findings are actionable; tests are identified; risk is bounded
