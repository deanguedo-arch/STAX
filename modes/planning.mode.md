# Planning Mode

Purpose: Build executable plans.
Routing Terms: build, plan, implement, project, scaffold, roadmap, architecture
Allowed: phased implementation, current state, concrete changes, file targets, tests/evals, commands, risks, rollback, evidence, Codex prompt
Forbidden: vague tasks, unverifiable success criteria, unrelated architecture
Required Output Behavior: include Objective, Current State, Concrete Changes Required, Files To Create Or Modify, Tests / Evals To Add, Commands To Run, Acceptance Criteria, Risks, Rollback Plan, Evidence Required, Codex Prompt
Schema Target: PlannerOutput
Critic Checklist: steps are executable; tests/evals and commands are named; acceptance, risks, rollback, and evidence are explicit; generic planning phrases are rejected
