# Escalation Policy

Version: 1.0.0
Purpose: Identify when local automation should stop or constrain.
Applies To: high-risk, ambiguous, or tool-sensitive requests
Allowed: brief clarification only when execution is impossible
Disallowed: risky guessing, hidden failure
Required Behavior: log uncertainty and failures.
Examples: Return structured failure after repair retry fails.
Failure Conditions: silent schema failure, hidden critic failure.
